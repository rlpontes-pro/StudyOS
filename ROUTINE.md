# StudyOS Routine

## Monday to Saturday

1. Study.
2. Update the current Week file.
3. Update `KNOWLEDGE/` (if you learned something worth keeping).
4. Commit.
5. Push.

### Git

```bash
git status
git add .
git commit -m "<message>"
git push
```

### Commit Message Pattern

#### Study

```text
study: complete Agent Skills module 2
study: finish CS50 Project 0
study: complete Week 01
```

#### Notes

```text
notes: update Anthropic concepts
notes: add Python notes
notes: refine RAG knowledge
```

#### Documentation

```text
docs: improve README
docs: update ROADMAP
```

#### Refactoring

```text
refactor: simplify StudyOS architecture
```

Avoid generic messages:

```text
update
changes
fix
misc
```

---

## Sunday

### Weekly Review

- Review the completed week.
- Complete the **Weekly Review** section.
- Update `KNOWLEDGE/` (if needed).
- Create the next Week file.
- Plan the next week's schedule.
- Commit.
- Push.
